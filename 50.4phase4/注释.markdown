3. 如果前端少传了字段，会报错吗？
------------------------------------------------------------
场景：Struct 有 A, B 两个字段，但前端只传了 {"a": "..."}。

- 默认行为 (Gin & 原生)：【不报错】。
  * 缺失的字段 B 会被赋予该类型的“零值”（字符串是 "", 整型是 0）。
  
- 如何让它报错？
  * 在 Struct 标签增加 `binding:"required"`。
  * 此时若缺了 B，ShouldBindJSON 会返回一个 non-nil 的 error。

- 如何区分“没传”和“传了空值”？
  * 使用指针：B *string `json:"b"`。
  * 若没传 B：变量 B 为 nil。
  * 若传了 {"b": ""}：变量 B 不为 nil，指向的内容是 ""。

4. ShouldBindJSON vs json.NewDecoder
------------------------------------------------------------
- ShouldBindJSON：Web 开发首选。一行代码搞定：`读取、解析、校验、报错`提示。
- NewDecoder：底层或非 Web 场景使用。纯粹的翻译官，`不带任何业务逻辑校验`。





Gin c.Abort() 函数深度解析笔记
1. 误区纠正

- 问：c.Abort() 会直接结束进程吗？
- 答：【不会】。它只是停止 Gin 处理链的后续流动。
- 问：c.Abort() 会像 return 一样结束当前函数吗？
- 答：【不会】。Abort 之后的代码依然会继续执行，除非你手动 return。




1. "Running in 'debug' mode..." (运行在调试模式)
